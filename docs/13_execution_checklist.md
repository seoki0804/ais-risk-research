# 문서명
오늘 바로 착수할 수 있는 실행 체크리스트

# 문서 목적
오늘 할 일, 이번 주 할 일, 첫 번째 마일스톤, 데이터 확보 전 검증할 가설, 초기에 만들 최소 산출물을 체크리스트 형태로 정리한다.

# 대상 독자
프로젝트 수행자, PM, 연구책임자

# 작성 버전
v1.0

# 핵심 요약
- [확정] 오늘의 핵심은 `데이터 소스 확인`, `대상 해역 후보 좁히기`, `전처리/feature 스켈레톤 만들기`, `baseline 파라미터 파일 만들기`다.
- [확정] 이번 주 목표는 `분석 가능한 trajectory segment 확보`와 `첫 heatmap 초안 출력`이다.
- [합리적 가정] 첫 번째 마일스톤은 `단일 사례에서 3개 speed scenario heatmap과 contour가 출력되는 노트북`이다.
- [확정] 데이터가 완벽하지 않아도 `가설 검증용 최소 산출물`부터 만드는 것이 중요하다.

## 1. 배경 및 문제 정의

문서가 정리되어도 실제 착수가 늦어지는 경우가 많다. 이 프로젝트는 데이터, 수식, 시각화가 서로 얽혀 있으므로 "오늘 무엇을 해야 하는가"가 분명해야 한다. 본 체크리스트는 바로 실행 가능한 수준으로 일을 쪼개어 제공한다.

## 4. 상세 설계/요구사항/방법론

### 4.1 오늘 할 일

- [ ] 공개 AIS 데이터 소스 후보 2~3개를 조사하고 라이선스를 체크한다.
- [ ] 대상 해역 후보를 2개 이내로 줄인다.
- [ ] raw field 기준 스키마 문서를 만든다.
- [ ] 폴더 구조를 만든다: `data/raw`, `data/curated`, `notebooks`, `src`, `configs`.
- [ ] baseline 파라미터 초안(`horizon`, `cell size`, `scenario multiplier`)을 config로 분리한다.
- [ ] trajectory reconstruction용 최소 스크립트 또는 노트북을 만든다.
- [ ] 자선 선정 기준 메모를 작성한다.

### 4.2 이번 주 할 일

- [ ] 샘플 AIS 파일 적재 및 컬럼 프로파일링
- [ ] 결측률과 gap 분포 분석
- [ ] track segmentation 및 interpolation 구현
- [ ] 상대거리, bearing, TCPA, DCPA 계산 함수 작성
- [ ] representative case 후보 5건 추출
- [ ] 첫 spatial grid와 contour 초안 렌더링

### 4.3 첫 번째 마일스톤

마일스톤 정의:

> 단일 실제 AIS 사례를 입력하면 자선 중심의 heatmap과 contour가 `현재 속력 / 20% 감속 / 20% 증속` 3개 시나리오로 출력되는 notebook을 완성한다.

완료 기준:

- [ ] 자선과 타선 상태가 올바르게 그려진다.
- [ ] heatmap이 최소 1개 사례에서 직관적으로 보인다.
- [ ] contour가 noisy하지 않게 출력된다.
- [ ] 주요 기여 factor를 텍스트로 출력할 수 있다.

### 4.4 데이터 확보 전 검증할 가설

| 가설 | 검증 방법 |
|---|---|
| 15분 horizon이 너무 길지 않은가 | synthetic encounter로 contour 확산 정도 확인 |
| 250m cell이 너무 조밀하거나 거칠지 않은가 | 100m/250m/500m 테스트 |
| max aggregation이 해석에 유리한가 | top-k sum과 결과 비교 |
| 0.35/0.65 threshold가 초기 시각화에 적절한가 | toy example 시각화 확인 |
| 3개 speed scenario만으로도 차이가 충분히 보이는가 | canonical case 생성 후 비교 |

### 4.5 초기에 만들 최소 산출물

| 산출물 | 목적 |
|---|---|
| EDA notebook 1개 | 데이터 이해 |
| pairwise feature notebook 1개 | 수학/도메인 검증 |
| risk heatmap notebook 1개 | 핵심 가치 검증 |
| representative case 이미지 3장 | 발표/포트폴리오 seed |
| config 파일 1개 | 파라미터 고정 |

### 4.6 착수 순서 권장

1. 데이터 소스/해역 확정
2. 전처리와 track reconstruction
3. pairwise feature 계산
4. rule-based severity
5. spatial projection + contour
6. 시나리오 비교
7. UI/ML 확장
