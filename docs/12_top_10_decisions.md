# 문서명
가장 먼저 확정해야 할 의사결정 10가지

# 문서 목적
프로젝트 초기에 반드시 선결정해야 하는 항목 10개를 우선순위와 함께 정리하고, 각 항목을 왜 먼저 정해야 하는지 설명한다.

# 대상 독자
PM, 연구책임자, 데이터사이언티스트, 지도교수

# 작성 버전
v1.0

# 핵심 요약
- [확정] 가장 먼저 확정해야 하는 것은 `데이터 소스/라이선스`, `대상 해역`, `baseline risk 정의`, `속력 시나리오`, `grid/horizon`이다.
- [확정] 이 10개가 정해지지 않으면 데이터 파이프라인, 실험, UI, 발표 모두 흔들린다.
- [합리적 가정] 초기 권장안은 `단일 혼잡 해역`, `4주 AIS`, `15분 horizon`, `250m grid`, `3개 속력 시나리오`다.
- [확정] ML 모델 선택은 선결정 항목이 아니라 후결정 항목으로 다루는 것이 타당하다.

## 1. 배경 및 문제 정의

프로젝트 실패의 흔한 원인은 중요한 의사결정을 너무 늦게 하는 것이다. 특히 AIS 기반 위험도 프로젝트는 데이터 소스, 해역, 위험 정의, 시나리오 설계가 먼저 잠기지 않으면, 후속 구현과 연구 문서가 연쇄적으로 수정된다. 따라서 선결정 10가지를 명시적으로 잠그는 것이 필요하다.

## 4. 상세 설계/요구사항/방법론

### 4.1 우선순위 기준

- P0: 미확정 시 전체 구조가 흔들리는 항목
- P1: 미확정이어도 일부 진행 가능하나 재작업 리스크가 큰 항목
- P2: baseline 이후 정해도 되는 항목

### 4.2 선결정 10가지

| 우선순위 | 의사결정 항목 | 권장 기본값 | 왜 먼저 정해야 하는가 |
|---|---|---|---|
| P0 | 1. 공개 AIS 데이터 소스와 라이선스 | 공개 이력 AIS + 연구/포트폴리오 사용 가능 소스 | 수집 가능성, 저장 방식, 공개 가능 범위가 달라진다 |
| P0 | 2. 대상 해역 | 혼잡한 단일 항만 접근 해역 1곳 | 교통 패턴, 데이터 규모, 사례 확보 난이도가 달라진다 |
| P0 | 3. 자선 정의 방식 | 특정 MMSI의 연속 track segment를 자선으로 선택 | 자선 중심 좌표계와 시나리오 분석 구조가 이 결정에 의존한다 |
| P0 | 4. baseline risk score 정의 | distance + DCPA + TCPA + bearing + encounter + density | 모든 실험과 UI 설명이 이 점수에 걸린다 |
| P0 | 5. speed scenario 세트 | 0.8x / 1.0x / 1.2x | UI, 실험, 결과 해석 프레임이 정해진다 |
| P0 | 6. horizon과 time step | 15분 / 30초 | CPA 계산, spatial projection, 계산량에 직접 영향 |
| P0 | 7. grid 반경과 cell 크기 | 6 NM / 250m | 해상도와 연산량, contour 품질을 좌우한다 |
| P1 | 8. pseudo-label 기준 | 향후 15분 내 realized minimum separation threshold | ML 비교안과 평가 지표가 여기에 의존한다 |
| P1 | 9. 시각화 제공 방식 | notebook 우선, web MVP 후속 | 일정 분배와 구현 우선순위가 달라진다 |
| P2 | 10. ML 범위 | Logistic Regression + Gradient Boosting까지만 | baseline 완료 전에는 결정해도 실익이 적다 |

### 4.3 결정 지연 시 영향

| 항목 | 지연 시 영향 |
|---|---|
| 데이터 소스/라이선스 | 전처리 파이프라인 재작성 가능성 |
| 대상 해역 | 사례 추출 로직과 density 기준 재설정 필요 |
| risk 정의 | heatmap과 contour 논리가 흔들림 |
| scenario 세트 | UI와 실험표 재설계 필요 |
| grid/horizon | 성능과 시각 품질 재조정 필요 |
