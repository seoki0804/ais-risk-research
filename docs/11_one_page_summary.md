# 핵심 요약
- [확정] 이 프로젝트는 공개 AIS만으로 `자선 주변 위험 공간`을 계산해 `heatmap`과 `safety contour`로 보여주는 시스템이다.
- [확정] 핵심 차별점은 `pairwise risk 판단`을 `own-ship-centric spatial risk mapping`으로 확장한 점이다.
- [확정] `현재 속력 / 감속 / 증속` 시나리오를 비교해 위험 분포가 어떻게 달라지는지 설명 가능하게 보여준다.
- [확정] 완전자율운항 제어가 아니라 `설명 가능한 항해 위험도 인지 및 의사결정 지원`이 목표다.

## 1. 배경 및 문제 정의

기존 AIS 기반 위험 분석은 주로 선박 간 개별 관계나 CPA/TCPA 수치에 집중한다. 하지만 실제 의사결정에서는 "자선 주변 어느 방향과 어느 공간이 더 위험한가"를 직관적으로 보는 표현이 더 중요하다.

## 4. 상세 설계/요구사항/방법론

### 문제

- pairwise 숫자는 공간 위험을 직관적으로 보여주지 못한다.
- 속력 변화가 위험도에 미치는 영향을 비교하기 어렵다.
- 해양 도메인에서는 설명 가능한 결과가 중요하다.

### 해결 방식

1. AIS 데이터로 자선/타선의 상대운동과 CPA/TCPA를 계산한다.
2. 자선 주변을 grid/cell로 나눈다.
3. pairwise severity를 공간 셀에 투영해 heatmap을 만든다.
4. 임계값 기반 contour로 위험/주의/안전 경계를 시각화한다.
5. 속력 시나리오(감속/현속/증속)를 비교한다.

### 차별점

| 차별점 | 설명 |
|---|---|
| Spatial risk mapping | pairwise 위험을 공간 분포로 확장 |
| Speed scenario comparison | 속력 변화에 따른 contour 변화 비교 |
| Explainable baseline | 규칙 기반 score로 결과를 설명 가능 |

### 핵심 화면/결과물

- 메인 지도: 자선 중심 heatmap + safety contour
- 비교 화면: 현재/감속/증속 3분할 view
- 설명 패널: 주요 위험 선박, TCPA/DCPA, 고위험 방향 설명

### 기술 포인트

| 항목 | 설명 |
|---|---|
| 데이터 | 공개 AIS, trajectory reconstruction |
| 위험도 | distance, DCPA, TCPA, bearing, encounter, density 기반 |
| 시각화 | heatmap, iso-risk contour |
| 확장 | Logistic Regression / Gradient Boosting comparator |
